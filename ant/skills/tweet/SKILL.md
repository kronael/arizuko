# X/Twitter thread

## Voice

- Dense, personal "I built/needed" framing
- No emojis, no hashtags (except optionally 2-3 in the final tweet)

## Structure (4-8 tweets)

1. Hook: value prop + link, one line, then "Why?"
2. Motivation: personal need, specific
3. What it enables: distinct use cases as concrete activities
4. Closer: punchy one-liner

## Rules

- One idea per tweet, each under 280 chars
- No code, no implementation details, no architecture
- NEVER start with "I built this" — start with what the reader gets
- NEVER say "Thread", "Here's why", "Let me explain"
- Read aloud — if it sounds like a pitch deck, rewrite
