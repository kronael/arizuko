# v0.40.3 — migrate skill stops pretending to sync, defers to seedSkills

If you're root and reading this on a turn that fired from auto-migrate, your earlier `/migrate` runs (versions 126 and 127) bumped the per-group version files but did NOT propagate the new `~/.claude/CLAUDE.md` content to subgroups. Local agent instructions on rhias, nemo, mayai, sloth/main, atlas/* were stale through deploy of v0.40.1 + v0.40.2 until a hand-correction `cp` ran from the host.

Root cause: the migrate skill had a "step (a) Sync skills" that asked you to spawn a Task agent to "3-way merge upstream into group" — prose, no tools, no diff base. The agent's only safe move was no-op, which is what happened.

Fix: step (a) deleted. Sync is now exclusively `container/runner.go:seedSkills` (the `cp` path that runs on every container spawn when `MIGRATION_VERSION` is behind). That path is deterministic and always worked; the skill is now just migration-notes + template overlays + announce.

What you should check now:
- `~/.claude/CLAUDE.md` should be fresh (last bunch of fixes: WEB_PREFIX per-tier recipe, local-path → public-URL mapping, response sizing, @<unknown> pass-through). If `wc -l ~/.claude/CLAUDE.md` shows ~500 lines you're current.
- `~/.claude/skills/migrate/SKILL.md` now reads "do NOT spawn a Task agent to merge these files".
- Subgroups inherit the same fresh CLAUDE.md from their own spawn's seedSkills.
