# 151 — v0.46.0: full-text search, message actions, reply-default, config-as-YAML

What's new for you:

## find_messages — search your whole history

`find_messages` does FTS5 full-text search over messages: bare tokens,
`"exact phrase"`, `a OR b`, `a NOT b`, `prefix*`, `NEAR(a b, 5)`.
Optional `scope` (a chat_jid or a folder subtree), `sender`, `since`,
`limit` (default 20, max 200). Ranked by bm25 with a snippet. Reach for
it instead of scrolling — "did anyone mention X", "what did I say about
Y last week", "where did we land on Z".

## Message actions: pin / unpin / edit / delete

- `pin_message` / `unpin_message` / `unpin_all` — pin a message to the
  channel (Slack / Discord / Telegram).
- `edit` — rewrite a message **you sent** in place (corrections, live
  status). Platform time-windows apply.
- `delete` — retract a message **you sent**.

Own-message by default; the platform enforces authorship, so trying to
edit/delete a user's message errors — don't retry. Adapters without the
primitive return an Unsupported hint — also don't retry.

## reply is the default; send is explicit

Use **`reply` for virtually every answer** — it threads under the
conversation you're answering (on Slack it lands in the thread, not the
channel root). Omit `replyToId` and it threads to the current
conversation automatically. Use **`send` ONLY** when you deliberately
need a fresh top-level message that is NOT a reply — a proactive
notification, or a message to a different chat. Posting an answer via
`send` drops it at the channel root, out of the thread.
