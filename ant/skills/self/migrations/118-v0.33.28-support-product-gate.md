# v0.33.28 — /support skill scoped to support product

The `/support` skill is now bound to support-product groups only. It
checks `~/PRODUCT.md` for `name = "support"` or `brand = "atlas"`
before invoking any phase. If the group is not the support product,
the skill exits immediately without dispatching its case/gather/reply/
verify/persist workflow.

Other agents (Rhias, content/sloth, krons/mayai, etc.) carry the
skill file but it no-ops for them. Dispatch sees the skill in the
description list but `/resolve` matches it weakly — the description
opens with "USE ONLY in groups deployed as the support product"
and includes "NOT for non-support groups (do not dispatch — return
immediately)" so the matcher avoids it.

Future: move `ant/skills/support/` → `ant/template/support/skills/`
once the per-product template overlay system (already specified in
`migrate/SKILL.md:94-148`) is exercised end-to-end. Today the
description-gate is the interim binding.
