# v0.35.0 — proxyd routes are TOML-declared

No skill changes. Release marker so auto-migrate fires and the v0.35.0
CHANGELOG blockquote is broadcast to all groups.

Channel adapters now declare their proxyd routes in
`template/services/<daemon>.toml` as `[[proxyd_route]]` blocks; the
compose generator collects survivors (after `gated_by` env filtering),
serializes them to JSON, and injects as `PROXYD_ROUTES_JSON` on proxyd.
No Go edits in `proxyd/main.go` or `compose/compose.go` when adding a
new adapter. Existing routes (`/slack/`, `/slink/`, `/chat/`, `/api/`,
`/x/`, `/static/`, `/mcp`, `/dash/`, `/dav/`, `/onbod/`) behave
identically.

Spec: `specs/6/2-proxyd-standalone.md`.
