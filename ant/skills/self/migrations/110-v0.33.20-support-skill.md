# v0.33.20 — /support skill

New `/support` skill orchestrates verified-answer responses with
primary-source citations and tracks multi-turn cases on the same
entity. Composes existing skills: `/recall-memories` and `/find` for
source discovery, `/facts` and `/users` for persistence.

When the user asks a concrete factual question about a domain entity
(validator, account, transaction, epoch, order, ticket), `/support`
runs four phases: **case** (continuation detection by entity ID,
reply-pointer, or correction phrase) → **gather** (open canonical
source, grep literal ID, read recorded outcome) → **reply**
(front-load answer + cite source path + field, render through
`~/SOUL.md` Voice) → **persist** (write new sources to
`~/facts/sources.md`, corrections to `/users`).

Range queries enumerate the full range — never sample one and
conclude. Continuations carry source path + prior cite; new entity
with no shared signal resets the case.

Spec: `specs/7/2-support-skill.md`. Used by the support product
(Atlas) and any agent answering domain questions in chat.
