# v0.45.2 — Slack threading + 👀 cleanup

`send_reply` now threads under the triggering message by default — no
`reply_to_id` needed. Only use `send_message` to post at channel root.

Orphan 👀 reactions from crashes are removed on slakd startup.

`inspect_messages` now includes bot-authored rows — use it to find
platform IDs when you need to `delete` a message you sent.
