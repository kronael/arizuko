# v0.40.6 — slack-team installable end-to-end

The slack-team product page promised composition + autoviv + email
quarantine. Hostile oracle found three broken commands that would
trip a fresh operator on first run. All fixed.

- `arizuko create acme --product slack-team` now lands a fully-seeded
  main group (PERSONA + CLAUDE + facts/). Previous setup.html step 1
  said `arizuko create acme` only — the product seed was invisible.
- `arizuko group <inst> add <jid> <folder> --product <name>` is real.
  cmdGroup now accepts `--product`, mirroring cmdCreate, so the
  compose-with-support pattern (`--product support` on a child group)
  actually seeds the support PERSONA + CLAUDE + facts. Without the
  flag, child groups got only the default agent.
- `arizuko create --help` lists slack-team in the product set. Was
  missing.
- Setup.html §12 `add_route` invocations now use the real
  `route:='<json>'` shape per ipc/ipc.go:1525. The flat
  `match:= target:= seq:=` form rejected with "route.target required".
- Setup.html §11 email ingest inlines `EMAIL_TRUSTED_AUTHSERV` +
  `EMAIL_TRUSTED_DOMAINS` + `EMAIL_STRICT_AUTH` (spec 8/17). The
  slack-team CLAUDE.md routes on `verb=untrusted`; without these set
  every inbound was untrusted and the quarantine path silently
  dominated.

No agent-side action needed. Operators creating new instances will
get the working flow from setup.html.
