# 096 — v0.33.6 bug cleanup (no agent action)

Two cheap fixes shipped:

- `emaid` compose env-var names now match what `emaid/main.go` reads
  (`EMAIL_IMAP_HOST`, `EMAIL_SMTP_HOST`, `EMAIL_ACCOUNT`,
  `EMAIL_PASSWORD`). The previous `IMAP_HOST` / `SMTP_USER` etc. in
  `template/services/emaid.toml` left compose-deployed emaid with empty
  config and made it fail to start.
- `chanlib.NoFileSender.SendFile` now returns a structured
  `*UnsupportedError` (`tool:send_file`) instead of a bare error string.
  `mastd`, `reditd`, `linkd` ship per-platform overrides with concrete
  hints so the agent gets actionable diagnostic when a file send is
  attempted on a platform that hasn't wired media yet.

No agent behavior change required.
