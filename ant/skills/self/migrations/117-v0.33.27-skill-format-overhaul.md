# v0.33.27 — skill format overhaul + 8 new skills harvested

**All skills now match Anthropic's canonical Claude Code format**:
single `description:` field carrying USE/NOT trigger phrases. The
`when_to_use:` field (silently ignored by arizuko's dispatch awk
loop) is dropped across 42 skills. Folded content into `description`.

**All skills now carry explicit `user-invocable: true`** — except
`dispatch` and `resolve` which stay `false` (internal only, run by the
gateway nudge / by /resolve respectively). Default-on means any user
can invoke any skill via slash command.

**Trigger phrases tightened** — descriptions now include 2-4 literal
phrases users type (`"second opinion"`, `"fix this Go function"`,
`"write a Dockerfile"`) plus 1-2 file types or contexts
(`.py files`, `Cargo.toml`, `scheduled task`, `*_test.go`). The
dispatch awk reads only `description:`; keyword density matters.

**8 new skills harvested from `kronael/tools`**:

- `distill` — recursive 5/3 summarization via @distill subagent
- `learn` — mine session history for new skills via @learn subagent
- `pr-draft` — draft PR description from staged changes
- `merge-trivial` — classify and resolve trivial git merge conflicts
- `release` — version bump + changelog + git tag pipeline
- `improve` — targeted code-quality fix via @improve subagent
- `readme` — sync README / ARCHITECTURE docs via @readme subagent
- `visual` — UI/CSS tweaks via headful browser via @visual subagent

**`wisdom/SKILL.md` body rewritten** to document the canonical
single-`description` shape, the USE/NOT pattern, and the
"user-invocable default" rule. Future skill authors apply both rules
without re-derivation.

**Flagged for operator review** (not changed in this release):
- `info` vs `self` — distinct now, but could collapse
- `soul` — deprecated stub; safe to delete on next cleanup
- `slink-mcp` vs `mcp` — disambiguated; no consolidation needed
