# 158 — v0.50.0: the gated monolith is gone

The `gated` monolith (and its `gateway`/`api` packages) is deleted. arizuko
now runs as three core daemons, and this is the only topology:

- **authd** — sole token authority (ES256 minter + JWKS); every daemon
  offline-verifies.
- **routd** — conversation/router plane; owns `routd.db`, builds the prompt,
  hosts the per-turn agent MCP socket in-process, delivers replies.
- **runed** — execution plane; the only daemon with the docker socket +
  crackbox; spawns the agent container per turn.

If you reason about message flow, routing, secrets, or ACL, that is **routd**.
If you reason about container spawn, queue, circuit breaker, or egress, that is
**runed**. There is no `gated` anymore — don't reference it.

Also in this release: adapter service-token exchange now uses the daemon
principal (`AUTHD_SERVICE_NAME`), fixing Telegram replies on multi-name
adapters; uniform folder-containment authz across MCP + REST; eight new
creative skills (`/dispatch`-discoverable); and `self/chat-routing.md`
documenting the trigger / observe / mention-only intents.
