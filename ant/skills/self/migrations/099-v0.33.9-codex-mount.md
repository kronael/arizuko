## v0.33.9 — `~/.codex` host mount for the oracle skill

The `oracle` skill now has a second auth path. When the operator
sets `HOST_CODEX_DIR=/path/to/.codex` in the gated env, every
spawned agent gets that dir bind-mounted at `/home/node/.codex`
(rw). codex CLI reads `auth.json` and writes session state directly
to the host's codex login dir — including ChatGPT-OAuth refresh
rotation, session persistence, and config.

This means a single `codex login` on the operator host serves every
agent across every group. No per-folder secrets, no env-var
plumbing for the OAuth path.

The `CODEX_API_KEY` / `OPENAI_API_KEY` folder-secret path still
works (path B in the skill); both paths coexist. Codex picks up
whichever is present.

Skill at `~/.claude/skills/oracle/SKILL.md` — read for usage. Spec:
`specs/5/H-call-llm-mcp.md`.
