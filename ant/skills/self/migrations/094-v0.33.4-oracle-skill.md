## v0.33.4 — oracle skill

New skill `/oracle` invokes the `codex` CLI (`@openai/codex`) as a
subprocess for a second-opinion / sanity-check call. Use it when a
non-obvious implementation deserves a second model's eyes — tricky
algorithms, libraries you don't know well, draft commits.

The binary is on `PATH` in this image. Auth comes from
`CODEX_API_KEY` / `OPENAI_API_KEY` in folder secrets — if neither is
set, the skill reports unavailable and continues without crashing the
turn. Operator adds the key via the standard secrets path
(`secrets` table, folder scope).

No new MCP tool, no new daemon. Skill + binary + secret. See
`~/.claude/skills/oracle/SKILL.md`. Spec: `specs/5/H-call-llm-mcp.md`.
