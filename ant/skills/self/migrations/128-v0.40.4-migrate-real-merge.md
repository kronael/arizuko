# v0.40.4 — migrate skill stops lying, does real 3-way merge

The migrate skill used to claim "3-way merge with conflict resolution"
in step (a). It had no merge base. The cheapest legal answer to "merge
without a base" was no-op, which is what happened — v0.40.1 + v0.40.2
content never propagated to subgroups until a hand-correction `cp`.

This release fixes the actual cause + ships the structural improvements:

- `container/seedSkills` now writes every synced file to two paths:
  `~/.claude/<path>` (the live file) AND
  `~/.claude/.merge-base/<path>` (the snapshot for next merge).
  Merge-base mirrors upstream exactly (renames + deletions propagate).
- On group create, `.claude/CLAUDE.md` + every stock skill seeded.
  On re-seed (rare), `ours/` is preserved (skip-if-exists) and only
  `.merge-base/` is refreshed. Operator edits to `ours/` survive.
- `~/.claude/skills/<name>/.disabled` sentinel: touch it to disable a
  stock skill. `seedSkills` removes its `SKILL.md` (Claude Code stops
  indexing it) AND wipes `.merge-base/skills/<name>/` (re-enable later
  starts fresh).
- migrate skill step (a) is now a real 3-way merge per file:
  - base missing → cp theirs to ours + base
  - theirs missing → flag DELETION in announce, leave ours intact
  - ours == base → operator hasn't touched; take upstream
  - theirs == base → upstream hasn't changed; preserve ours
  - both changed → agent reads base/ours/theirs inline, writes merge
- Bash hardened: `set -euo pipefail`, `shopt -s nullglob`, `find -print0`
  + `while IFS= read -r -d ''`.
- `gateway.checkMigrationVersion` keeps the child-notify backstop so a
  failed `/migrate` doesn't silently hide release news from subgroups.
- New ownership convention: `<group>/CLAUDE.md` is operator-owned (ant
  NEVER touches); `<group>/.claude/CLAUDE.md` is ant-managed via merge.
  Customize at the group level. Documented in `ant/CLAUDE.md` Storage.

Reading this on the auto-fire after v0.40.4 deploy: your skill set is
already fresh because seedSkills ran before this turn. Verify with
`wc -l ~/.claude/CLAUDE.md` (~500 lines) and `cat ~/.claude/skills/self/MIGRATION_VERSION` (== 128).
