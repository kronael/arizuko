# v0.40.5 — emaid auth, dashd refresh, slack-team template, teled fix

A grab-bag release tightening operator-facing surfaces.

- emaid spec 8/17 shipped: DMARC + sender-allowlist + verb=untrusted annotation. `EMAIL_TRUSTED_AUTHSERV` env required (fail-closed default); operator pins the trusted upstream MX, all other Authentication-Results headers dropped per RFC 8601 §5. `EMAIL_STRICT_AUTH=true` (also accepts 1/yes/on) drops untrusted mail outright. Gateway spec 6/J amended: verb=untrusted never promotes to mention.
- dashd visual refresh: AWS-like sans body, single accent (blue), sticky table headers, zebra rows, `.btn-danger` class. Path-aware nav with aria-current; breadcrumbs on group settings / new-group / whatsapp-pair pages.
- slack-team product template completed at `ant/examples/slack-team/`: PRODUCT.md (manifest with required env hints), PERSONA.md (channel-aware voice), CLAUDE.md (group overlay with autoviv + email-quarantine instructions), facts/README.md. `arizuko create acme --product slack-team` now actually does something.
- teled: drop self only (not every third-party bot). DeeperDexBot postings in observed channels now reach gateway.
- Dockerfile: emaid, bskyd, mastd, reditd binaries added to image (were missing — adapters couldn't actually start).
- specs/8/18 (discussion): AWS-Console per-daemon /dash/ + central hub pattern. Audit identifies shared chanlib template as the biggest win (one change → 8 adapter dashboards). Implementation deferred to v0.41.
