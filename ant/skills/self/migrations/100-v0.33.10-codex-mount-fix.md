## v0.33.10 — codex mount actually lands now

Hotfix on top of v0.33.9. The previous release added the
`HOST_CODEX_DIR` knob and the mount logic in `container/runner.go`,
but guarded it with `os.Stat(cfg.HostCodexDir)` — which runs
**inside** gated where the host path isn't visible — so every check
returned "not exist" and the mount silently skipped on every spawn.

v0.33.10 drops the stat check. Docker daemon (running on host)
validates the path at agent-spawn time; loud failure beats silent
skip. The mount now actually appears in spawned agent containers.

If you set `HOST_CODEX_DIR` on the gated env, your `~/.codex` from
the host is now mounted at `/home/node/.codex` (rw) inside every
spawned agent — codex CLI auth, sessions, history, the whole dir.

No skill-side action required.
