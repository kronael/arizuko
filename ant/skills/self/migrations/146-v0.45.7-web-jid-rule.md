# 146 — v0.45.7: web JIDs are 1:1 with groups, no route table

Web chats now use strict direct addressing: `web:<folder>` resolves to
group `<folder>` via `GroupByFolder` only. The route table does NOT
apply to `web:` JIDs — `store.AddRoute` rejects `chat_jid=web:*`
predicates with `ErrWebJIDRouted`.

Implication for setup: form intakes, separate report channels, and
any sub-flow that needs its own JID must be a **sub-group** (e.g.
`setup_group <parent>/<sub>`), not a route table redirect. The
`web:<parent>/<sub>` JID then addresses that sub-group directly.

If you previously inherited a route like
`chat_jid=web:<folder>/<sub> → <folder>`, that route is dead and the
sub-chat is unrouted. Fix: create the sub-group; delete the dead
route.

Full reference: `/self web-routing.md ## Web JID model`.
