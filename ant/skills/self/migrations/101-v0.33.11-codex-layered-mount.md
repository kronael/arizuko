## v0.33.11 — codex mount layered, isolation per group

The `HOST_CODEX_DIR` mount (v0.33.9/10) gave every group rw access
to a single shared codex state dir. That meant agents could read
each other's codex sessions/history/memories. Closed today.

New shape:
- Per-group writable `.codex/` at `<groupDir>/.codex` mounted to
  `/home/node/.codex` (rw). Isolated workspace per group: history,
  sessions, memories, sqlite state. No cross-group leak.
- Host's `auth.json` and `config.toml` from `HOST_CODEX_DIR`
  bind-mounted RO **on top** of the per-group dir at
  `/home/node/.codex/auth.json` and `/home/node/.codex/config.toml`.
  Shared credentials, never written by agents — refresh-token
  rotation stays in-memory per-process.

Operator one-time setup: `chmod 644 ~/.codex/auth.json
~/.codex/config.toml` so the agent's uid 1000 can read mode-600
host files.

End-to-end e2e harness landed in `webd/slink_e2e_test.go` —
`make test-e2e` runs slink-driven scenarios; `make test` skips
them via `testing.Short()`. CI gates e2e on release tags.

No skill-side action.
