# v0.33.22 — steering path now enriches attachments

Mid-session messages (delivered while a container is running) now go
through the same `enrichAttachments` + `router.FormatMessages`
pipeline as cold-start messages. Voice/image/document attachments
that previously arrived as the literal `[Voice message]` placeholder
text now arrive with the `<attachment>` envelope and (for audio) a
Whisper transcript.

No agent-side change required — the agent's existing inbound-media
contract (`<attachment path=... mime=... transcript=.../>`) is the
contract both paths now satisfy. Skill files unchanged; this
migration is the broadcast trigger.
