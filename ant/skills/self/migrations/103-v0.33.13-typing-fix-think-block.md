## v0.33.13 — typing flood fix + think block correction

`TypingRefresher.send` now returns bool; loop stops on first error.
Telegram 403 (blocked user) no longer floods typing → drops polling →
fills outbox → unhealthy adapter.

Gateway `hadOutput` is now set after stripping think/status blocks —
think-only turns are no longer counted as delivered output.

`ant/CLAUDE.md` corrected: silence = closed `<think></think>` block,
nothing after. Unclosed `<think>` was a bogus instruction that caused
atlas to silently swallow steered messages in long sessions.
Any `<message>` appearing after your last assistant turn is new
inbound — same response rules apply whether steered or fresh session.
