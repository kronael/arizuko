# v0.39.0 — route mode via URI fragment

`routes.target` now takes an optional `#mode` fragment. Two values:

- `folder` (no fragment) — **trigger**. Inbound stores normally and the
  agent fires a turn. Today's default, unchanged.
- `folder#observe` — **observe**. Inbound stores under `folder` with
  `is_observed=1` but does NOT fire a turn. You see observed messages
  via the existing folder ACL (`inspect_messages`, `get_history`) and,
  on the next trigger turn, as a trailing `<observed>` block in the
  prompt envelope.

The `routes.impulse_config` column is gone. Verb filtering is now an
explicit `verb=<v>` match key + `seq` priority — the canonical
mention-only channel is a `verb=mention` trigger row stacked above a
catch-all `#observe` row.

Migration 0054 auto-converts existing impulse-config routes; operators
don't touch anything. New observe-only routes use
`target = folder#observe`. Window defaults:
`OBSERVE_WINDOW_MESSAGES=10`, `OBSERVE_WINDOW_CHARS=4000`; per-route
overrides on `routes.observe_window_messages` /
`routes.observe_window_chars`.

When you see an `<observed>` block, treat it as context only — do not
reply to it; reply to the explicit message. The gateway emits this
rule in the system prompt automatically when observed messages are
present.
