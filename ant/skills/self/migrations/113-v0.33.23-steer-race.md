# v0.33.23 — steer-into-dying-container race fixed

When a chat message arrives while a container is in the act of
exiting, gated previously believed the queue slot was still active
(no exit event yet from docker), wrote the message to the IPC input
dir, sent SIGUSR1 to a dead process, and treated the steer as
successful. The chat cursor advanced, the IPC file was orphaned, and
no future poll picked the message up until the user sent something
else.

Fix: gated now captures the SIGUSR1 error. If the signal fails the
slot is marked inactive and `SendMessages` returns false; the caller
falls through to `EnqueueMessageCheck`, which spawns a fresh
container that drains the orphan via `drainIpcInput()` at session
start.

No agent-side change. This migration is the broadcast trigger.
