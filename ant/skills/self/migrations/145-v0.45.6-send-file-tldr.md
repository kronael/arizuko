# 145 — v0.45.6: send_file always includes TL;DR caption

When delivering a file via `send_file`, always distill the file's key
findings into a 2-4 sentence TL;DR and pass it as the `caption`
parameter. The user gets the file and understands it without opening it.

No separate `send`/`reply` after `send_file` — the caption is the summary.
