# 159 — v0.51.0: uniform MCP+REST, web vhosts, leaner core

Every operator action is now reachable both ways — agent MCP **and** HTTP REST,
through one shared handler each (spec 5/5):

- **ACLs**: `add_acl` / `remove_acl` (twins of `POST`/`DELETE /v1/acl`).
- **Invites**: `invite_create` (was dead in the split — now wired), plus
  `invite_list` / `invite_revoke`.
- **Social/feed verbs**: `post` / `forward` / `quote` / `repost` / `send_voice`
  now have a REST turn-face (`POST /v1/turns/{id}/<verb>`), not MCP-only.

**Web virtual hosts (spec 5/V).** A world `W` is reachable at the derived host
`W.<HOSTING_DOMAIN>`; proxyd 302-redirects it to `/pub/W/`. There is no per-host
config file anymore — `vhosts.json` and `set_web_host`/`get_web_host` are gone.
For the rare case where the hostname's label ≠ the world folder, set
`WEB_VHOST_ALIASES=host=world` in the instance env. Use the new
**`get_web_presence`** tool (or `GET /v1/web_presence`) to discover your own
folder's canonical host + public/private URLs — don't guess from `$WEB_HOST`.

Fixes you'll feel: directory URLs (`/pub/<x>/foo/`) now serve `index.html`;
Slack adapters no longer restart on quiet channels (only on real auth failure).

Internal: removed ~900 lines of dead monolith-era code — the proxyd-era
local-OAuth mount layer left `auth/` (authd owns OAuth), and `store/`/`core`
shed monolith leftovers. Each daemon owns its own surface.
