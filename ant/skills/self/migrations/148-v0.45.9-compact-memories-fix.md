# 148 — v0.45.9: compact-memories correctness fixes

Tightens the compact-memories skill after a three-instance audit
(krons + marinade + sloth) surfaced silent-success bugs: 578 cron
runs marked `status=success` while >35 week/month rollups were
silently skipped despite source files on disk; `diary/month/`
directory never created in any group across any instance.

Changes to `~/.claude/skills/compact-memories/SKILL.md`:

- **Deterministic target-period math.** Day = yesterday UTC.
  Week = last completed ISO week. Month = last completed calendar
  month. Explicit `date -u +%G-W%V` reference. Was: "yesterday UTC"
  for day only, implicit for week/month — produced flaky week-label
  bugs (e.g. atlas/2026-W22.md mislabeling current week).
- **Optional period override arg** for backfills:
  `/compact-memories episodes week 2026-W18` targets W18
  instead of last-completed. Auto cron omits the arg.
- **Compact-log JSONL.** Every invocation now writes one line to
  `~/episodes/.compact-log.jl` or `~/diary/.compact-log.jl` with
  outcome (WRITE / SKIP_NO_SOURCES / SKIP_EXISTS /
  SKIP_VALIDATION_FAIL), sources_found count, output path.
  `task_run_logs.status='success'` only confirms dispatch; the
  compact-log is the actual audit trail.
- **Diary-month path explicit.** Skill must `mkdir -p ~/diary/month/`
  before writing — directory did not exist in any group across any
  instance, blocking 100% of diary-month rollups.
- **Strict file-naming + frontmatter validation.** Period format
  must match level (`YYYYMMDD` / `YYYY-WNN` / `YYYY-MM`). Sources
  must be bare filenames — no `episodes/` prefix. No duplicates.
  Self-check before write; fail → log SKIP_VALIDATION_FAIL.
- **Don't recompact existing files** unless override arg given.
  Was: implicit "skip" — sometimes interpreted as overwrite, causing
  re-compaction smell with stale `aggregated_at` stamps.
- **Silent-by-convention reinforced.** Cron compactions emit no chat
  output, no `<status>` blocks. The diff (file or not) + the
  compact-log line are the artifacts.

Triggers for retroactive backfill of missed periods (W17–W21,
April month, March month) using the new override arg. Operators
can run e.g. `/compact-memories episodes week 2026-W19` per
affected group.

No schema changes. No daemon changes. Skill-only.
