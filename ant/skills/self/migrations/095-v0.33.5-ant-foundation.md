# 095 — v0.33.5 — ant package skeleton

The `ant/` directory now hosts a standalone-agent Go package alongside
the existing TS runtime. Foundation only: nothing in the running
container changes.

What landed:

- `ant/cmd/ant/main.go` — CLI flag stub (`<folder> [--prompt=...]
  [--mcp [--socket=...]] [--sandbox=none|dockbox|crackbox]`). Body
  exits `64` (EX_USAGE); only `--help` exits 0.
- `ant/pkg/agent/loader.go` — folder-layout resolver
  (`SOUL.md`, `CLAUDE.md`, `skills/`, `diary/`, `secrets/`, `MCP.json`,
  `workspace/`). Tested.
- `ant/pkg/host/`, `ant/pkg/runtime/` — package stubs (sandbox + Claude
  CLI driver, both deferred).
- `ant/scripts/curate-skills.sh` — portable vs arizuko-only partition
  gate. Current count: **37 portable, 1 arizuko-only** (`self`). No
  skills moved this pass.

The runtime port (replacing `ant/src/index.ts`) and the
`ant-arizuko/` skill split are tracked in
`specs/5/b-ant-standalone.md` and not yet shipped. The TS path and
`arizuko-ant:latest` image are unchanged — keep using them as before.
