# v0.33.24 — self-loop fix + status-without-conclusion contract

Two related platform fixes plus an agent-contract rule.

**Gateway: self-routed local dispatch skipped.** When the agent's
reply chat_jid is a local folder AND matches the agent's own
groupFolder, the redundant `LocalChannel.Send` is skipped. The
outbound row was already authoritative; the duplicate persist as
synthetic inbound spawned another agent run that replied to its
own echo (witnessed sloth/main 2026-05-10 21:12–21:14, 5
self-acknowledgments in 51s). Cross-group sends (different folder)
still flow through normally — that's legit escalation.

**Migrate skill: broadcast scoped to root group only.** Previously
`/migrate` fanned out the version announcement to every group's
primary JID, including operator DMs mid-conversation (witnessed
marinade 2026-05-10 11:44 interleaving a v0.33.22 broadcast into a
"cool" reply flow). Now announces to the current root folder's JID
only.

**CLAUDE.md: status-without-conclusion rule.** If you emit a
`<status>` block, you owe a final user-visible reply. Silent tasks
(file writes, cron compactions) emit neither status nor final text.
Status without conclusion is a contract break — gateway can't
rescue it without becoming brittle, so the agent must follow the
rule.
