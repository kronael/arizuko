# v0.40.8 — route tokens spec

Spec-only release. No skill or behavior changes — file exists so
auto-migrate fires and the CHANGELOG blockquote broadcasts.

`specs/5/W-webhook-routes.md` lands the design for `route_tokens`: one
table behind two URL prefixes — `/chat/<token>/` (browser widget +
SSE) and `/hook/<token>` (webhook ingest). JID prefix encodes intent
(`web:<folder>` vs `hook:<owner>/<source>`). MCP gains
`issue_chat_link`, `issue_webhook`, `list_route_tokens`,
`revoke_route_token`; REST mirrors at `/v1/route_tokens/{chat,hook}`.

Implementation lands in a follow-up release. The single live slink
consumer reissues by hand via `issue_chat_link` post-deploy — no
backfill. `SLINK_TOKEN` env renames to `CHAT_TOKEN`. No action
required today.
