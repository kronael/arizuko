# 155 — v0.49.0 — rendering fixes, output-style refresh, docs rebuild

Mostly under-the-hood. One thing you'll notice: your replies render and thread
better, and per-surface style edits now reach you on every turn. No action
required.

## What changed (agent-relevant)

- **Telegram formatting fixed**: `**bold**`, `#` headings, and `` `code` ``
  render correctly now (teled converts your markdown to Telegram HTML — keep
  emitting markdown, never raw HTML; post bare URLs, not `[text](url)`).
- **Output styles refresh every spawn**: per-surface tone/length/format files
  are re-seeded into your group on every turn, so platform-style fixes take
  effect without a re-onboard.
- **Thread resume**: only valid session UUIDs are resumed; arizuko session
  placeholders start fresh — Slack/Discord threads stop occasionally restarting.

## What changed (operator/platform, not agent)

- **Daemon split flipped**: routd hosts the per-turn agent MCP socket in-process;
  runed is now a pure container-spawner (federation removed). Behind
  `CUTOVER_SPLIT` — dormant; default deployments still run the gated monolith.
- **routd correctness** (dormant): breaker no longer trips on clean runs,
  round_done wired, steered turns finalized, dispatch forwards
  model/container-config/isolation, double-persist removed, GetRoute 404-vs-500,
  route-token JID validation, egress allowlist + grants wired to runed.
- **Docs**: repo docs, per-package READMEs, specs, and the reference site
  reconciled to the running code (schema 0071, MCP tiers, split env vars).
