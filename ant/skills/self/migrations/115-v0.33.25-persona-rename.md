# v0.33.25 — SOUL.md renamed to PERSONA.md + gateway per-turn anchor

The group-level persona file is now `PERSONA.md` (uppercase, at group
root). Term aligns with universal LLM lexicon — agent recognizes it.

**Gateway prepends a `<persona>` summary on every inbound turn**,
extracted from `PERSONA.md` frontmatter `summary:` field. Re-anchors
voice register without depending on the system-prompt alone (which
drifts in multi-turn sessions). If `summary:` is missing the block is
empty — strict frontmatter, no fallback.

**New `/persona` skill** re-reads the full PERSONA.md on demand. The
gateway-injected summary is cheap (every turn); the full re-read is
on-demand only.

**Schema modeled on elizaOS Character format** with arizuko adaptations:
added `summary:` for the cheap per-turn injection tier, kept `lore:`
as a separate field from `bio:`, dropped infra fields (`plugins`,
`settings`, `secrets`, `templates`). Canonical fields: `name`,
`summary`, `system`, `bio[]`, `lore[]`, `adjectives[]`, `topics[]`,
`style.all[]`, `style.chat[]`, `messageExamples[]`, `knowledge[]`.

Existing `<group>/SOUL.md` files auto-migrate to `PERSONA.md` on next
container spawn (`container/runner.go` renames the legacy file in
place).

The `/soul` skill is deprecated and now redirects to `/persona` for
read-only refresh. Authoring is operator-only — they edit the file
directly.

Migrate broadcast scoping (v0.33.24) reverted in this release —
announcements fan out to all group JIDs again, per operator request.

Reference: `refs/eliza/packages/shared/src/onboarding-presets.characters.ts`
(elizaOS canonical character format, post-v1).
