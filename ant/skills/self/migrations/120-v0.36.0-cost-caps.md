# v0.36.0 — cost caps

Spec 5/34 lands: per-folder daily budget caps + automatic Anthropic
usage capture + `log_external_cost` MCP tool for oracle/codex.

What this means for your skills:

- **Anthropic spend is tracked automatically.** Every turn submits
  per-model token usage via the existing `submit_turn` envelope.
- **`/oracle` now reports its codex cost.** The skill instructs you
  to call `codex --json exec` and pass token counts + cost_usd to the
  new `log_external_cost` tool. Old `codex exec "..."` calls
  still work but go uncounted toward cost-caps.
- **The pre-spawn gate refuses turns over cap.** Operators set caps
  with `arizuko budget <inst> set folder <name> --daily N`. When a
  channel exceeds today's cap, the next inbound message gets a
  short "Budget reached for today" reply without an LLM call.

Default cap is 0 (uncapped) on every folder — existing deployments
are unaffected until an operator opts in.

Spec: `specs/5/34-cost-caps.md`.
