# v0.41.0 — route tokens: chat links and webhooks

Schema migration `0059-route-tokens.sql` creates the `route_tokens` table and
drops the `groups.slink_token` column.

New MCP tools: `issue_chat_link`, `issue_webhook`, `list_tokens`, `revoke_token`.
Use these to mint public chat URLs and webhook endpoints on demand. The raw token
is returned exactly once — store it in your workspace file.

URL surfaces: `/chat/<token>/` (widget + SSE) and `/hook/<token>` (ingest).
Old `/slink/*` URLs 301-redirect to `/chat/*`.

The operator panel moved from `/chat/<folder>` to `/panel/<folder>`.

CLI: `arizuko token issue|list|revoke`. Dashboard: `/dash/tokens/<folder>/`.

Reference: `chat-link.md` in this skills package.
