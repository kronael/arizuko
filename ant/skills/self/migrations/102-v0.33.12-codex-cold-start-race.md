## v0.33.12 — codex per-group dir cold-start race fixed

v0.33.11's per-group `.codex/` mount worked fine on warm
single-spawns but lost a race on cold start: when gated booted
and auto-migrate fired spawns for many groups in parallel, the
runner's lazy `os.MkdirAll(<groupDir>/.codex)` was sometimes
beaten by `docker run`, which then materialized the bind source
as root. The agent (uid 1000) couldn't write codex state and
`codex login status` failed with `Permission denied`.

Fix: gated now pre-seeds every group's `.codex/` synchronously
at startup, before `checkMigrationVersion` enqueues any
auto-migrate spawn. Runs as gated's uid 1000, so the dirs land
with correct ownership.

The runner's spawn-time `os.MkdirAll` stays as a safety net.

Operator one-time on each instance: chown any pre-existing
root-owned `.codex/` dirs left over from v0.33.11:

```bash
sudo find /srv/data/arizuko_<inst>/groups -mindepth 2 -maxdepth 3 \
  -name .codex -user root -exec chown -R 1000:1000 {} \;
```

No skill-side action.
