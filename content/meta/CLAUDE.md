# content/meta/ — Structure

Foundation docs for all arizuko content. Read before drafting any post.
This directory governs the content/ tree only — not the codebase.

## Files

- `pillars.md` — 3 core claims every post traces back to, ordered by priority
- `voice.md` — persona, writing rules, register; what we never say
- `editorial.md` — evidence standards, non-negotiables
- `audience.md` — who reads this, how to write for them
- `method.md` — article types, pipeline phases, spec file format, build order

## Reading order

Before writing any post: pillars → voice → editorial → audience → method
