How arizuko thinks. One page per primitive: what it is, why it exists, what you need to know to use it.

entries

  • ant — an agent defined by a folder; persona, skills, memory, secrets in one place
  • slink — public HTTP endpoint per group; chat in a browser, speak the protocol from a script, or talk to it as MCP
  • jid — typed addresses for chats and senders, the same shape across every platform adapter
  • routing — one table maps inbound JIDs to ant folders; supports globs, reply-chain, sticky topics
  • grants — what an agent can do, composed at spawn from path-depth tier + folder rules + user ACL
  • auth — OAuth + local login + account linking; collision UX, single canonical sub
  • webdav — per-group file workspace over WebDAV; mount it from Finder, rclone, or curl
  • voice — Whisper in, Kokoro out; agent decides when to speak, the platform delivers a real voice note
