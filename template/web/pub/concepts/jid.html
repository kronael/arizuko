jid

typed addresses for chats and senders · ← concepts

Every chat and every sender has a typed address — <platform>:<kind>/<id> — so a routing rule like chat_jid=telegram:group/* matches all Telegram groups regardless of platform-specific quirks.

why

Each platform names things its own way. Telegram chat IDs are signed integers where positive means DM and negative means group — so old rules carried a sign-bit hack. Reddit shoves comments, users, and submissions behind t1_/t2_/t3_ prefix soup on the same reddit:. WhatsApp suffixes a server (@g.us, @s.whatsapp.net) to mean group vs DM. And ChatJID and UserJID were both string in Go — the compiler couldn’t catch a swap.

One JID = one resource. The kind goes in the wire form, the type goes in the code, and routes match with plain glob semantics.

show me

telegram:user/123              # telegram DM with user 123
telegram:group/-100456         # telegram group/supergroup/channel
discord:dm/456                 # discord DM channel
whatsapp:789@g.us              # whatsapp group (server distinguishes kind)
reddit:comment/abc             # reddit comment
mastodon:account/42            # mastodon account
web:slink/<token>              # anonymous web chat via slink

# routes match with path.Match globs (* never crosses /)
match='chat_jid=telegram:group/*'    # all telegram groups
match='chat_jid=discord:dm/*'        # all discord DMs
match='chat_jid=whatsapp:*@g.us'     # all whatsapp groups
match='sender=telegram:user/[0-9]*'  # any numeric telegram user

what you get

  • Kind in the path. First segment after the colon is the discriminator. No sign-bit tricks, no prefix soup — telegram:user/ is a DM, telegram:group/ is a group, the wire form says so.
  • Typed Go values. core.ChatJID and core.UserJID are distinct types over *url.URL. The compiler refuses to swap a chat for a sender.
  • Plain glob routing. path.Match semantics — * matches any non-/ sequence, ? one char, […] a class. Same rules across chat_jid, sender, platform, verb.
  • Hard cutover already done. Migration 0038-typed-jids.sql rewrote every JID-shaped value in the store — messages, chats, user_jids, grants, onboarding, and the predicates inside routes.match. Idempotent.
  • Per-platform schemas owned by adapters. Core treats <rest> as opaque except for glob semantics. Each adapter declares its own shape and parses its own paths — adding a kind is a one-string change there, no system-wide format bump.
  • URI-clean. Built on net/url, so percent-encoding for IDs with reserved chars (Bluesky DIDs contain :) is free.
