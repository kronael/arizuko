voice

concepts / voice · ← concepts

Send a voice message — Whisper transcribes it into the agent’s prompt. Reply with send_voice — Kokoro synthesizes audio and the platform delivers a real voice note.

why

Users who send voice expect voice replies. Whisper has handled the inbound side for a while; send_voice closes the loop on the outbound side. The two are symmetric: voice in is just text-in to the agent, voice out is just text the agent decides to speak. The agent picks the modality — voice-first personas, matching the user’s channel, or staying silent — the system doesn’t auto-convert anything.

voice in (Whisper)

Set WHISPER_BASE_URL and VOICE_TRANSCRIPTION_ENABLED=true. Inbound voice notes are downloaded, transcribed, and surface in the agent’s prompt as an attachment with the transcript inline:

<attachment path="..." mime="audio/ogg" transcript="...">

The agent reads the transcript like any other text. The audio file is on disk if it ever needs the raw bytes.

voice out (send_voice)

MCP tool: send_voice(chatJid, text, voice?). Distinct from send_file because each platform’s voice primitive is distinct from its file/audio primitive (Telegram sendVoice vs sendAudio; WhatsApp ptt:true vs plain audio).

Voice resolution chain: explicit voice arg > voice: frontmatter in SOUL.md > TTS_VOICE env. Gateway hashes sha256(text+voice+model), checks <data_dir>/tts/<hash>.ogg, and on miss POSTs to an OpenAI-compatible /v1/audio/speech at TTS_BASE_URL — default points at the bundled ttsd daemon, a thin Go proxy in front of Kokoro-FastAPI. Operators who prefer Piper, Coqui, or OpenAI cloud override TTS_BACKEND_URL and skip the bundled service. Text capped at 5000 chars.

what you get

  • Symmetric voice in/out. Whisper for inbound, Kokoro for outbound — one mental model on both sides.
  • Local TTS by default. Bundled ttsd proxies a Kokoro-FastAPI container; no cloud round-trip, no per-character billing.
  • Operator-swappable backend. Point TTS_BASE_URL at any OpenAI-compatible /v1/audio/speech endpoint — Piper, Coqui, OpenAI cloud, anything.
  • Per-agent voice. voice: in SOUL.md frontmatter pins one ant’s voice; TTS_VOICE sets the instance default.
  • Cached output. Same text + voice + model hits disk on the second send — no re-synth.
  • Native push-to-talk where it exists. Telegram and WhatsApp render as PTT; Discord renders inline audio; unsupported platforms get chanlib.ErrUnsupported and the agent falls back to plain send.
