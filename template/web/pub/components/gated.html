gated (removed)

Removed. gated was the original monolith daemon. It was split into three daemons and deleted in v0.50.0 (2026-06-07). The split is now the only topology. This page stays so old links don’t break; the content below points you to where each job moved.

What it was

gated was one Go binary that did everything: it held the message database, ran the poll loop that turned inbound rows into agent runs, served the HTTP API channel adapters registered against, hosted the per-group MCP sockets agents talked back through, spawned the per-turn container, and checked authorization. One process, one schema, every job.

That worked, but it braided unrelated concerns into a single failure domain — a routing bug and a container-spawn bug lived in the same binary. The split pulls those apart into three daemons that each own one job and one database.

Where each job went

  • routd — the routing and conversation plane. Owns routd.db (messages, routes, groups, acl, secrets, scheduled tasks). Runs the poll loop, resolves each inbound row to a group, hosts the per-turn agent MCP socket in-process, and delivers replies back to the channel adapters. Adapters POST here; the grant/ACL authority lives here too. See concepts/routing.
  • runed — the execution plane. The only daemon wired to the Docker socket: it spawns and tears down the per-turn arizuko-ant container. Owns runed.db (spawns, session log).
  • authd — authentication and identity. Mints and verifies tokens, runs the OAuth provider routes, owns auth.db. It fetches the login-time scope ceiling from routd (the ACL owner). See concepts/auth.

The single messages.db is gone — each daemon owns and migrates its own SQLite file. The one-struct, four-endpoints pattern (REST + MCP + YAML manifest + OpenAPI) that gated used for its config rows survives unchanged in routd; see reference/openapi.
