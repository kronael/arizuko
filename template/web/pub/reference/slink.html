slink

arizuko › reference › slink

Public per-group web channel: one token, one POST endpoint, one round-handle protocol. Drops a message into the group's inbox and exposes the agent's reply as a snapshot, status probe, or SSE stream. Spec: specs/1/W-slink.md; implementation in webd/slink.go, webd/turn.go, webd/slink_mcp.go.

Token

Format
32 random bytes, base64url-encoded (~43 chars, 256 bits).
Storage
Column groups.slink_token; resolved via store.GroupBySlinkToken on every /slink/* hit.
Lifetime
Minted once at group registration. Not rotated, not expired.
Visibility
Public — embedded verbatim in page source, links, scripts, MCP client config.
Security model
Treat a leaked token like a leaked rate-limit bucket, not like a leaked password. Defence is rate limiting (see below) + grant scope of the bound group, not token secrecy.

Rate limits

Enforced at the edge in proxyd/main.go:510 (dispatchSlink). Excess returns HTTP 429.

CallerBucket keyDefaultEnv var
Anonymous (no JWT)client IP10 req/minSLINK_ANON_RPM
Authenticated (JWT)JWT sub60 req/minSLINK_AUTH_RPM

Both buckets use a sliding 1-minute window. A valid JWT promotes the caller to the authenticated tier; otherwise the anonymous tier applies. Operators and agents reach gated and the MCP socket directly without going through /slink/*, so they don't hit either tier.

Sender identity

Derived in webd/slink.go:240 on every POST /slink/…:

  • Valid JWT — sender = X-User-Sub, sender_name = X-User-Name (proxyd-signed identity headers; auth.VerifyUserSig must pass).
  • No Authorization header — sender = anon:<ip-hash> (first 4 bytes of SHA-256(X-Forwarded-For)), sender_name = "Anonymous".
  • Malformed JWT — rejected upstream by proxyd with 401.

HTTP endpoints

Every route registered in webd/server.go:62 under the slinkCORS wrapper. {token} is the slink token; {id} is a turn_id (= the originating inbound messages.id).

POST shape

Request body (one of):

content=<text>&topic=<optional>       # application/x-www-form-urlencoded
{"content":"...","topic":"..."}      # application/json

Content negotiation on Accept:

  • Default (no Accept or application/json) — JSON, fire-and-handle.
  • text/html — HTMX bubble fragment (used by the built-in /chat page only).
  • text/event-stream — legacy SSE upgrade keyed on (folder, topic); preserved for chat-widget compat.

Default JSON response:

{
  "user": { "id": "msg_abc", "content": "...", "created_at": "..." },
  "turn_id": "msg_abc",
  "status": "pending"
}

turn_id == user.id for the first message of a round. Optional ?wait=<sec> (capped 120 s) blocks the response until the first assistant frame and adds "assistant": {…}.

Steer (POST /slink/{token}/{id}): same body, same status code. When the parent round is still pending, the response includes "chained_from": "{id}" and the message is queued as the next round for the folder. When the parent is already done, the steer becomes a fresh round and chained_from is omitted. The Claude Agent SDK cannot inject mid-turn input — "steering" is semantically “cut to the front of the next-turn queue”.

GET snapshot & cursor

GET /slink/{token}/{id}
GET /slink/{token}/{id}?after={msg_id}
{
  "turn_id": "msg_abc",
  "status": "pending|done|failed",
  "frames": [
    {"id":"out_001","content":"...","created_at":"...","kind":"message","turn_id":"msg_abc"},
    {"id":"out_002","content":"...","created_at":"...","kind":"status","turn_id":"msg_abc"}
  ],
  "last_frame_id": "out_002"
}

frames[] is ordered by created_at ascending. Each id is a stable messages.id; pass it as ?after={id} on the next call to fetch only what's new.

kind:

  • message — the agent's reply.
  • status — interim <status> block from the agent's status protocol (any frame whose content begins with ⏳ ; see turn.go:27).

Stop polling once status == "done". "failed" carries an error string. Hard cap: 200 frames per snapshot call.

GET status

GET /slink/{token}/{id}/status
{
  "turn_id": "msg_abc",
  "status": "pending",
  "frames_count": 2,
  "last_frame_id": "out_002"
}

Use this to decide between “poll once and stop” vs “open SSE” without paying for the frame payload.

GET SSE stream

GET /slink/{token}/{id}/sse

Events:

  • event: message / event: status — payload is one frame object (same shape as snapshot frames).
  • event: round_done — fires exactly once after submit_turn. Payload {turn_id, status, error?}. The server then closes the stream.

Last-Event-Id reconnect is supported; on reconnect the server replays frames with id greater than the last seen id before resuming live. Keep-alive ping (: ping) every 15 s.

/config JSON shape

From webd/slink.go:58. No auth (token in URL is the credential).

{
  "token": "646a...",
  "folder": "acme/support",
  "name": "Support",
  "endpoints": {
    "post":   "/slink/{token}",
    "stream": "/slink/{token}/{turn_id}/sse",
    "status": "/slink/{token}/{turn_id}/status"
  },
  "sdk": "/assets/arizuko-client.js"
}

Pages that don't bake paths at build time call /config first, then drive the rest of the protocol from the returned URLs. The sdk field lets cross-origin pages discover and import() the SDK at runtime.

<script src="/assets/arizuko-client.js"></script>
<script>
  const slink = await Arizuko.connect("<token>");
  const turn  = await slink.send("hello");
  slink.stream(turn.turn_id, {

    onStatus:  s => console.log("status", s),

CORS

Every /slink/* response (including /config, the snapshot endpoints, SSE, and MCP) carries:

Access-Control-Allow-Origin:  *
Access-Control-Allow-Methods: GET, POST, OPTIONS
Access-Control-Allow-Headers: Accept, Content-Type

Set by slinkCORS in webd/slink.go:438; preflight OPTIONS short-circuits to 204. Scope is /slink/* and /assets/* only — nothing else in webd is affected.

This is safe because: (1) the token is already public — CORS adds no new trust surface; (2) proxyd strips inbound identity headers on the way in, so cross-origin pages can't elevate to JWT identity via this path; (3) the rate-limit bucket is keyed on IP regardless of origin.

MCP transport

POST /slink/{token}/mcp exposes a standards-compliant Streamable HTTP MCP endpoint (webd/slink_mcp.go). Token possession is auth — same rate-limit tier as the form-encoded path. External MCP clients register the URL and call the group's three tools as local tools.

ToolParamsEffect
send_messagecontent (req), topic?Start a fresh round. Returns {turn_id, topic, folder}. turn_id is the originating messages.id — the round handle.
steerturn_id (req), content (req)Extend a round. While the round is pending, the follow-up reuses the same conversation thread; once done, starts a fresh round chained from this one. Returns {turn_id, topic, folder, chained_from?}.
get_roundturn_id (req), wait?Read assistant frames for this round (turn-scoped per messages.turn_id). With wait=true, blocks up to 5 min for an assistant reply. Returns {frames, done}.

Design rationale (round handle, three-tool surface, statelessness): specs/1/W-slink.md §MCP transport.

Schema dependencies

  • groups.slink_token — per-group token column.
  • messages.turn_id (migration 0038, indexed) — every outbound assistant message carries the inbound message id that triggered it. Snapshot/SSE page exactly by foreign key.
  • turn_results(folder, turn_id) (migration 0036) — one row per completed round; consumed by snapshot and status.

See reference/schema for column-level detail.
