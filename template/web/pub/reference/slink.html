slink

arizuko › reference › slink

Developer reference for the slink HTTP surface. One token per group, one POST to send, one round-handle to read. Spec: specs/1/W-slink.md. Code: webd/slink.go, webd/turn.go, webd/slink_mcp.go.

Token

Format
32 random bytes, base64url-encoded (~43 characters, 256 bits of entropy).
Storage
groups.slink_token. Resolved via store.GroupBySlinkToken on every /slink/* request.
Lifetime
Minted once when the group is created. Not rotated, not expired.
Visibility
Public. It appears verbatim in page source, links, and MCP client config.
Trust model
A leaked token is a leaked rate-limit bucket, not a leaked password. Defence is rate limiting plus the group’s grants.

Rate limits

Anonymous callers go through the DoS shield in proxyd/main.go (dispatchSlink). Excess returns HTTP 429. No X-RateLimit-* headers are emitted. Signed-in callers skip the shield; their spend is governed by the per-folder cost caps in specs/5/34-cost-caps.md.

Sender identity

Computed on every POST in webd/slink.go:

  • Valid JWT — sender = X-User-Sub, sender_name = X-User-Name. proxyd-signed; auth.VerifyUserSig must pass.
  • No Authorization header — sender = anon:<ip-hash> (first 4 bytes of SHA-256 of X-Forwarded-For), sender_name = "Anonymous".
  • Malformed JWT — rejected upstream by proxyd with 401.

HTTP endpoints

Registered in webd/server.go, all wrapped by slinkCORS. {token} is the slink token; {id} is a turn_id (the originating inbound messages.id).

MethodPathPurpose
GET/slink/{token}301 to /slink/{token}/chat
GET/slink/{token}/chatBuilt-in chat page (HTML)
GET/slink/{token}/configJSON bootstrap: token, folder, name, endpoints, SDK URL
POST/slink/{token}Send a message. Returns turn_id.
GET/slink/{token}/{id}Snapshot: status + all frames; ?after={msg_id} for cursor pages.
GET/slink/{token}/{id}/statusStatus + counts, no frame payload.
GET/slink/{token}/{id}/sseLive SSE; closes on round_done.
POST/slink/{token}/mcpStreamable HTTP MCP endpoint — three tools.

POST shape

Request body, either form or JSON:

content=<text>&topic=<optional>       # application/x-www-form-urlencoded
{"content":"...","topic":"..."}      # application/json

The response depends on the Accept header:

  • Default or application/json — JSON envelope (below).
  • text/html — HTMX fragment (used by the built-in /chat only).
  • text/event-stream — legacy SSE upgrade; chat-widget compat.

Default JSON response:

{
  "user":    { "id": "msg_abc", "content": "...", "created_at": "..." },
  "turn_id": "msg_abc",
  "status":  "pending"
}

turn_id == user.id for the first message of a round. Add ?wait=<sec> (capped 120 s) to block until the first assistant frame; the response then includes "assistant": {…}.

To continue a thread, POST again with the same topic. If the group’s container is still running, the new message is delivered mid-loop (queue.SendMessages). Otherwise a fresh round spawns.

GET snapshot

GET /slink/{token}/{id}
GET /slink/{token}/{id}?after={msg_id}
{
  "turn_id": "msg_abc",
  "status":  "pending|done|failed",
  "frames": [
    {"id":"out_001","content":"...","created_at":"...","kind":"message","turn_id":"msg_abc"},
    {"id":"out_002","content":"...","created_at":"...","kind":"status","turn_id":"msg_abc"}
  ],
  "last_frame_id": "out_002"
}

Frames are sorted by created_at ascending. Pass the last id as ?after={id} on the next call to fetch only what is new. Stop when status == "done". A "failed" response carries an error string. Hard cap: 200 frames per call.

kind:

  • message — the agent’s reply text.
  • status — interim status frame; content starts with ⏳ . See turn.go.

GET status

GET /slink/{token}/{id}/status

{
  "turn_id":       "msg_abc",
  "status":        "pending",
  "frames_count":  2,
  "last_frame_id": "out_002"
}

Cheap probe; no frame payload. Use it to decide between “poll once more” and “open SSE.”

SSE stream

GET /slink/{token}/{id}/sse

Event types:

  • event: message — one frame object (same shape as snapshot frames).
  • event: status — one status frame.
  • event: round_done — fires once. Payload {turn_id, status, error?}. Server then closes.

Reconnect is supported via Last-Event-Id: the server replays frames newer than the supplied id before resuming live. Keep-alive ping (: ping) every 15 s.

/config

From webd/slink.go. No auth — the token in the URL is the credential.

{
  "token":  "646a...",
  "folder": "acme/support",
  "name":   "Support",
  "endpoints": {
    "post":   "/slink/{token}",
    "stream": "/slink/{token}/{turn_id}/sse",
    "status": "/slink/{token}/{turn_id}/status"
  },
  "sdk": "/assets/arizuko-client.js"
}

Pages that do not bake paths at build time call /config first and drive the rest of the protocol from the returned URLs.

CORS

Every /slink/* response carries Access-Control-Allow-Origin: *, Allow-Methods: GET, POST, OPTIONS, Allow-Headers: Accept, Content-Type. Set by slinkCORS in webd/slink.go; OPTIONS short-circuits to 204. Scope: /slink/* and /assets/* only.

Safe because the token is already public, proxyd strips inbound identity headers, and the rate-limit bucket keys on IP.

MCP transport

POST /slink/{token}/mcp is a Streamable HTTP MCP endpoint (webd/slink_mcp.go). Token possession is the auth. Three tools mirror the REST surface one-to-one.

ToolParamsEffect
send_messagecontent (req), topic?Same as POST /slink/{token}. Reusing a topic continues that thread. Returns {turn_id, topic, folder}.
get_roundturn_id (req), after?, wait?Same as the snapshot GET. wait=true blocks up to 5 min for the next assistant frame. Returns {turn_id, status, frames, last_frame_id, done}.
get_round_statusturn_id (req)Same as the status GET. Returns {turn_id, status, frames_count, last_frame_id}.

One handler per action, two transports (REST and MCP), one audit correlator (slink.<action>). Drift is structurally precluded. Spec: specs/6/5-uniform-mcp-rest.md.

JavaScript SDK

Hosted at /assets/arizuko-client.js from a baked-in embed.FS (webd/assets.go). ETag content-hash, Cache-Control: public, max-age=3600, CORS open. Spec: specs/1/Z2-slink-sdk.md.

Schema

  • groups.slink_token — per-group token.
  • messages.turn_id (migration 0038, indexed) — every outbound assistant message carries the inbound id that triggered it. Snapshot and SSE page by this column.
  • turn_results(folder, turn_id) (migration 0036) — one row per completed round.
