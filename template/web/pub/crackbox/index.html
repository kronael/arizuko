crackbox

a sandbox proxy for untrusted code · ← hub

Run an AI agent, a CI script, or any other untrusted code with a tight allowlist of hosts it can reach. Everything else is silently blocked.

why

Every AI agent and CI script wants the network. You want it to reach github.com and api.anthropic.com — not evil.com, not your internal services, not random crypto miners shipped in a compromised npm package.

Browser permission prompts, iptables-per-process, and egress firewalls outside the host either don’t fit the shape or don’t know who is calling. crackbox is a tiny proxy you put in front of the workload and tell it what hosts are allowed.

30 seconds

docker pull crackbox:latest

# allowed → 200
crackbox run --allow github.com -- curl -s -o /dev/null -w '%{http_code}\n' https://github.com

# unlisted → blocked
crackbox run --allow github.com -- curl -s -o /dev/null -w '%{http_code}\n' https://example.com

# python image, install one package, nothing else
crackbox run --image python:3 --allow pypi.org,files.pythonhosted.org -- pip install requests

Subdomains match the parent (--allow github.com covers api.github.com). Comma-separated for multiple hosts. --allow "" blocks everything.

good for

  • AI agents. An agent that can curl the open internet is one prompt-injection away from talking to whoever the attacker wants. Pin it to the few hosts it actually needs.
  • CI & build steps. “npm install only talks to the registry” is a sentence you can mean.
  • Dev tools and demos. Run a tutorial repo, an unfamiliar binary, an LLM-generated script — with the assurance that whatever the workload does, it can’t exfiltrate to anywhere unexpected.

two shapes

For one-off runs, crackbox run --allow X -- <cmd> is a single command. It spins up a private Docker network, runs the proxy on it, runs your workload, tears everything down on exit.

For service deployments where many short-lived workloads come and go, run the proxy as a long-lived daemon and register each workload with it as it starts. See the reference for the daemon API, configuration file, and the rest.

install

docker pull crackbox:latest

One image, ~15 MB. Linux only. Open source — see the GitHub repository for source, issues, and the changelog.
