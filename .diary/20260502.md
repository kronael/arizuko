## ~08:05 — krons deploy: regression visible in chat

Operator caught it in chat: "wtf you sent invites to groups already
registered". Tracing it back yielded two distinct cascading bugs:

1. **Typed-JID migration tail missing.** `0042-typed-jids` rewrote
   JID-shaped values in 6 tables but missed three where they also
   live: `routes.match` `room=` predicates,
   `scheduled_tasks.chat_jid`, and `chat_reply_state.jid`. Effect:
   every routed telegram chat became unrouted because
   `JidRoom("telegram:user/<id>")` returns `"user/<id>"` while the
   route still matched on the bare ID. `gateway.resolveGroup`
   failed → `gateway.go:450` ran `InsertOnboarding` → `onbod` saw
   the `awaiting_message` rows in its 30s sweep and dispatched
   `/onboard?token=…` auth links to chats that have been
   operational for weeks. The "invites" were not the migrate
   skill's CHANGELOG broadcast (that step ran on each agent
   internally and emitted "v0.32.2 already announced"); they were
   onbod re-prompts of orphaned/unrouted JIDs.

2. **Marinade atlas stuck.** `arizuko_gated_marinade`'s container
   env still had `IDLE_TIMEOUT=60000` from the pre-`/strip` deploy
   (env file on disk was stripped today; container env is set at
   container creation, not at file change). `core.envDur` parses
   60000 as 60 ms, so the default-30min ratchet cut to 60s and
   killed the agent mid-turn after a single ack. Stripping `.env`
   on disk is necessary but insufficient — the running container
   needs a recreate to pick it up.

## ~09:00 — fix + deploy

`0043-typed-jids-tail.sql` rewrites all three missed columns using
the same kind-discriminator semantics as `0042`. Idempotent. Tested
on disk-copies of all three instances before pushing — confirmed
`room=` only rewrites for telegram (not whatsapp `@server` or web
`main`) by gating on `substr(...) NOT GLOB '*[^0-9]*'`. Note: SQLite
GLOB negation is `[^...]`, not `[!...]` (cost a dry run).
