Message Flow

// Channel → store → gateway poll → queue → container → channel

Channel.OnMessage → store.PutMessage + PutChat
  → gateway.messageLoop (polls every 2s)
  → store.NewMessages (unprocessed since cursor)
  → checkTrigger (direct mode or @name regex)
  → handleCommand (/new, /ping, /chatid, /stop)
  → router.ResolveRoutingTarget (delegate to child if matched)
  → queue.EnqueueMessageCheck → processGroupMessages
    → router.FormatMessages (XML batch)
    → container.Run (docker run -i --rm)
    → stream output → router.FormatOutbound
    → channel.Send

Each agent runs in its own docker container. The gateway mounts group folders, IPC dirs, and session state. Agent reads prompt from stdin, writes JSON to stdout. Signal-driven IPC (SIGUSR1) for immediate wake, 500ms fallback poll.

V1: What's Shipped

Go rewrite of the Kanipi TypeScript gateway. Same architecture, 17 packages, ~9,800 LOC. Five channels, container isolation, 14 IPC actions, permission tiers, scheduled tasks, circuit breaker, MCP sidecars, diary memory, skills system.

V1 Features
1
Container isolation
Every agent runs in its own docker container. No shared process space. 8 volume mount types with security validation. Skills seeded on first spawn. Timer-based timeout with graceful stop then kill. Session persistence across runs. Output delimited by ---NANOCLAW_OUTPUT_START--- / ---NANOCLAW_OUTPUT_END--- markers. Docker-in-docker path translation via HOST_DATA_DIR / HOST_APP_DIR.
2
Five channel adapters
Telegram via go-telegram-bot-api (long polling). Discord via discordgo (WebSocket gateway). WhatsApp via whatsmeow (Signal protocol). Email via IMAP IDLE + SMTP threading. Web SSE server-sent events broadcast hub. Each implements the same Channel interface. Enabled by config presence. Forward/reply metadata per channel. Media placeholders. Trigger translation.
3
Permission tiers
4-tier model derived from folder depth: root (tier 0, no / in folder), world (tier 1, one segment), agent (tier 2), worker (tier 3). Enforced on all IPC actions. Root gets full access. World gets rw on shared state. Agents get rw workdir with ro setup files. Workers get everything ro. Mount access varies by tier. inject_message restricted to tier 0/1. Group registration restricted to same-world parents.
4
Action registry (14 actions)
Unified action system with JSON schema validation. Self-registering init() pattern. Messaging: send_message, send_file, inject_message. Tasks: schedule_task, pause_task, resume_task, cancel_task. Groups: register_group, delegate_group, escalate_group, set_routing_rules, refresh_groups. Session: reset_session. Authorization enforced per action — agents can only act on their own group. Root vs non-root access control.
5
Scheduled tasks with CtxMode
SQLite-backed scheduler: cron, interval, and once modes. Cron via robfig/cron parser. Two context modes: group (injects prompt as user message, runs in shared group session) or isolated (fresh session per run, no persistence, results injected as context on next group message). Run log with per-task status and result. NextRun computed at task start to prevent re-scheduling while container runs.
6
Signal-driven IPC
File-based request/reply. Agent writes requests/<id>.json, gateway dispatches by type, writes replies/<id>.json. Atomic via tmp+rename. SIGUSR1 wakes agent immediately after gateway writes IPC file. 500ms fallback poll. Legacy fire-and-forget retained. Stdin piping via input/ dir for live message injection. Non-root groups restricted to registered JIDs.
7
Hierarchical routing
5-tier routing rules: command, pattern, keyword, sender, default. Parent groups delegate to children. Same-world authorization, direct parent-child only. Max delegation depth 3. Regex pattern capped at 200 chars (ReDoS mitigation). Groups registered via CLI with folder mapping. First group defaults to root. Subsequent groups use trigger mode (agent responds only when mentioned).
8
MCP sidecars
Per-group MCP server configuration in GroupConfig.Sidecars. Sidecar containers run alongside agent. Socket transport at ipc/sidecars/<name>.sock. Resource limits (memory, CPUs). Gateway manages full lifecycle: start, health check, settings wiring, cleanup. Socket path wired into agent settings.json as mcpServers entry.
9
Circuit breaker + error notifications
Per-group consecutive failure counter. After 3 failures, circuit opens — no more automatic retries. New user message resets the breaker. Agent errors advance the cursor and notify the user to retry manually. On error with no output: cursor rolled back, session evicted, messages retry. On error with output: cursor advances, partial work preserved.
10
Diary memory
Agent-written daily notes in group/diary/YYYYMMDD.md. YAML frontmatter summary injected on session start (two most recent files, with relative time: "today", "yesterday", "3 days ago"). Returned as XML annotations in <knowledge layer="diary"> tag.
11
MIME pipeline
Magic-bytes detection for 12 formats: JPEG, PNG, GIF, WEBP, AVI, WebM, MP4, OGG, FLAC, MP3, PDF, ZIP. RIFF subtype detection. Extension fallback. No external dependencies. Pluggable handler interface for attachment processing.
12
Mount security
Additional mounts validated against ~/.config/arizuko/mount-allowlist.json. Paths must be absolute, resolve symlinks, exist on host, be under an AllowedRoot. Blocked patterns: .ssh, .gnupg, .env, credentials, private keys. Non-root groups forced read-only. Container path: /workspace/extra/<name>. The allowlist itself is never mounted (tamper-proof from agents).

Workspace Layout

Each agent container receives a structured workspace namespace. Access varies by permission tier.

MountPathAccess
group/workspace/groupcurrent group workdir (rw, ro for workers)
media/workspace/mediamedia files (rw)
self/workspace/selfarizuko source (ro, root only)
sharegroups/<world>/share/cross-group shared state (rw root/world, ro agent/worker)
sessiondata/sessions/<folder>/.claude/agent session, skills, CLAUDE.md (rw, ro for tier 2+)
ipcdata/ipc/<folder>/IPC dirs: requests, replies, input, sidecars (rw)
web/workspace/webweb output (rw, root/world only)
extra/workspace/extra/...allowlisted additional mounts (ro, validated by mountsec)

/workspace/self is root-only (tier 0). Root and world get full rw on shared state. Agents get rw workdir with ro setup files. Workers get everything ro. All paths go through container.hp() for docker-in-docker translation.

Skills System

container/skills/ is seeded to ~/.claude/skills/ inside each container on first spawn for a group. Includes arizuko skills (self, migrate) plus development skills. A CLAUDE.md is seeded alongside. SOUL.md for per-group agent identity (personality, tone, style).

Migration System

container/skills/self/MIGRATION_VERSION tracks applied version. Numbered files in container/skills/self/migrations/ describe changes. The /migrate skill syncs all groups from canonical source when version advances. Gateway prepends annotation when group skills are behind current version.

Mythology

蟻塚女 — mistress of the ant hill. Ants embody organized persistence: each carries its fragment without understanding the whole, and the colony builds anyway. The mistress does not command — she coordinates by structure: routing rules determine which ant receives the work, permission tiers determine what each ant can touch, the circuit breaker determines when to stop and wait. Every session a grain is added. Every diary entry is the colony remembering. Every ant is expendable; the hill is not.

Channel Adapters

Telegram shipped
Long polling via go-telegram-bot-api. Markdown to HTML conversion.
JID: telegram:<chatID>. @bot mention translation. Media placeholders (photo, video, voice, sticker, document, location, contact). File sending routed by extension. 4096 char limit with HTML retry fallback. Forward/reply metadata.
Discord shipped
WebSocket gateway via discordgo. Intent-based event handling.
JID: discord:<channelID>. Bot mention <@id> to @name translation. Attachment type detection. Reply context fetch. 2000 char message splitting. Guild/channel name resolution.
WhatsApp shipped
Signal protocol via whatsmeow. Event-driven with offline queue.
JID: whatsapp:<jid>. SQLite auth store. Outgoing queue drains on reconnect. Forward/reply from contextInfo. Media: photo, video, voice, audio, document, sticker. File upload via whatsmeow.
Email shipped
IMAP IDLE inbound, SMTP outbound. Thread tracking via In-Reply-To.
JID: email:<threadID>. TLS on port 993. Exponential backoff reconnect (1s to 60s). SHA256 thread ID generation. Headers injected as context (From, Subject, Date).
Web SSE shipped
Server-sent events broadcast hub. Per-group listener management.
JID: web:<group>. Buffered channels (cap 16). Non-blocking send (drops if full). Thread-safe add/remove/disconnect.

Packages

gateway
Main loop, message routing, gateway commands.
Polls for new messages, dispatches to queue, handles /new /ping /chatid /stop. Per-chat error tracking with cursor rollback. Diary annotations injected into agent context.

Docker spawn, volume mounts, sidecars, skills seeding.

Serializes agent invocations per group. Global concurrency limit (default 5). Circuit breaker opens after 3 consecutive failures, reset by new message.

ipc
File-based IPC watcher. Request/reply + legacy fire-and-forget.

V2: Roadmap

V1 serves the operator. V2 adds audience-facing features, protocol upgrades, and new channels.

Topics + Sessions
1
#topic routing
Route messages within a group to different agents by topic or thread. Topic ID as JID segment. Glob pattern matching. Same group, different conversations, different handling.
2
Session resume tokens
Embed session ID in reply footer, extract on next message, pass --resume <id> for explicit continuation. Thread-level granularity within groups — the takopi pattern.
Memory Layers
1
Diary (shipped in V1)
Daily markdown notes with YAML frontmatter. The raw input layer. Two most recent files injected on session start with relative time labels.
2
Work (current task state)
Ephemeral work.md describing active task, blockers, next steps. Overwritten each time, not appended. Most volatile layer.
3
Episodes (weekly/monthly)
Scheduled aggregation of diary entries into episode summaries. Compresses daily notes into longer-term patterns and milestones.
4
Facts (permanent)
Conceptual knowledge distilled from episodes. Entities, relationships, recurring patterns.
Protocol + Channels
1
IPC to MCP migration
Replace file-based IPC with MCP over Unix socket. Structured protocol, bidirectional, typed tool calls. File IPC remains as fallback.
2
Credential injection via MCP proxy
Agent calls a tool, gateway adds the API key before forwarding. The credential never enters the container's process memory. Requires network isolation (--network none) to prevent exfiltration.
3
Web dashboard
Observe all agents, their state, costs, token usage, message flow. Built on the same IPC/MCP bus as everything else.
4
X/Twitter adapter
OAuth 2.0, rate-limited posting. Middleware intercepts post tool calls into a pending queue. No draft publishes without human approval.
V2 is designed but not implemented. Components will crystallize from V1 usage, not from spec documents.

Agent context

ARIZUKO is a multitenant Claude agent gateway written in Go. Evolved from Kanipi (TypeScript). V1.0.0 ships 5 channels: Telegram (go-telegram-bot-api), Discord (discordgo), WhatsApp (whatsmeow), Email (IMAP IDLE + SMTP), Web SSE. 17 packages, ~9,800 LOC Go, 122 tests. 4-tier permission model: root (tier 0), world (tier 1), agent (tier 2), worker (tier 3). Tier derived from folder depth. Mount isolation: 8 mount types with per-tier access control. Config: .env + env vars (no YAML). Channels enabled by token/config presence. 14 IPC actions with JSON schemas: messaging (send_message, send_file, inject_message), tasks (schedule/pause/resume/cancel), groups (register, delegate, escalate, routing_rules, refresh), session (reset). Authorization enforced per action with tier-based access. inject_message: insert into DB without channel delivery, tier 0/1 only. Scheduled tasks: cron/interval/once with two CtxModes — group (shared session) or isolated (fresh session, results injected on next message). MCP sidecars: per-group sidecar config, gateway-managed lifecycle, Unix socket transport. Signal-driven IPC: SIGUSR1 immediate wake, 500ms fallback. Atomic file writes via tmp+rename. Circuit breaker: 3 consecutive failures opens, new user message resets. Error notifications to users. Diary memory: daily markdown with YAML frontmatter, 2 most recent injected on session start. MIME pipeline: magic-bytes detection for 12 formats, pluggable handler interface. Mount security: allowlist validation, blocked patterns (.ssh, .gnupg, .env, credentials, private keys). Workspace: group(rw), media(rw), self(ro root-only), share(rw root/world, ro agent/worker), session(rw, ro tier 2+), ipc(rw), web(rw root/world), extra(ro allowlisted). Skills seeded from container/skills/ per group. SOUL.md for agent identity. Migration system: MIGRATION_VERSION + migrations/ dir; /migrate skill syncs groups. SQLite WAL, schema v2, 12 tables, 5s busy timeout. Docker-in-docker path translation. Hierarchical routing: 5-tier rules (command/pattern/keyword/sender/default), max depth 3. V2 planned: topic routing, session resume tokens, memory layers (diary/work/episodes/facts), IPC-to-MCP migration, credential injection via MCP proxy, web dashboard, X/Twitter adapter. Deploy: systemd per instance, arizuko CLI for create/config/group management.
