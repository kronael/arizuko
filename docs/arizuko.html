Message Flow

// Channel → store → gateway poll → queue → container → channel

Channel.OnMessage → store.PutMessage + PutChat
  → gateway.messageLoop (polls every 2s)
  → store.NewMessages (unprocessed since cursor)
  → checkTrigger (direct mode or @name regex)
  → handleCommand (/new, /ping, /chatid, /stop)
  → router.ResolveRoutingTarget (delegate to child if matched)
  → queue.EnqueueMessageCheck → processGroupMessages
    → router.FormatMessages (XML batch)
    → container.Run (docker run -i --rm)
    → stream output → router.FormatOutbound
    → channel.Send

Each agent runs in its own docker container. The gateway mounts group folders, IPC dirs, and session state. Agent reads prompt from stdin, writes JSON to stdout. Signal-driven IPC (SIGUSR1) for immediate wake, 500ms fallback poll.

Channel Adapters

Telegram shipped
Long polling via go-telegram-bot-api. Markdown→HTML conversion.
JID: telegram:<chatID>. @bot mention translation. Media placeholders (photo, video, voice, sticker, document, location, contact). File sending routed by extension. 4096 char limit with HTML retry fallback. Forward/reply metadata.
Discord shipped
WebSocket gateway via discordgo. Intent-based event handling.
JID: discord:<channelID>. Bot mention <@id> → @name translation. Attachment type detection. Reply context fetch. 2000 char message splitting. Guild/channel name resolution.
WhatsApp shipped
Signal protocol via whatsmeow. Event-driven with offline queue.
JID: whatsapp:<jid>. SQLite auth store. Outgoing queue drains on reconnect. Forward/reply from contextInfo. Bot detection (own number vs shared). Media: photo, video, voice, audio, document, sticker. File upload via whatsmeow.
Email shipped
IMAP IDLE inbound, SMTP outbound. Thread tracking via In-Reply-To.
JID: email:<threadID>. TLS on port 993. Exponential backoff reconnect (1s→60s). SHA256 thread ID generation. Headers injected as context (From, Subject, Date). Dependency-injected thread storage.
Web SSE shipped
Server-sent events broadcast hub. Per-group listener management.
JID: web:<group>. Buffered channels (cap 16). Non-blocking send (drops if full). Thread-safe add/remove/disconnect.

Packages

gateway shipped
Main loop, message routing, gateway commands.
Polls for new messages, dispatches to queue, handles /new /ping /chatid /stop. Per-chat error tracking with cursor rollback. Diary annotations injected into agent context.
container shipped
Docker spawn, volume mounts, sidecars, skills seeding.
Builds 8 mount types (group, media, self, share, session, IPC, web, extra). Seeds settings.json with env vars and MCP sidecar config. Manages sidecar container lifecycle.
queue shipped
Per-group concurrency, stdin piping, circuit breaker.
Serializes agent invocations per group. Global concurrency limit (default 5). Circuit breaker opens after 3 consecutive failures, reset by new message.
router shipped
XML message formatting, 5-tier routing rules.
Command → pattern → keyword → sender → default. Parent-child delegation within same world, max depth 3. Outbound filtering strips internal tags.
store shipped
SQLite persistence. 12 tables, WAL mode, 5s busy timeout.
Messages, chats, groups, sessions, session log, system messages, scheduled tasks, task run logs, router state, auth, email threads.
actions shipped
Unified action registry. JSON schemas, authorization.
init() self-registration. Actions: send_message, send_file, reset_session, schedule_task, pause/resume/cancel_task, register_group, delegate_group, set_routing_rules, refresh_groups. Root vs non-root authorization.
channels shipped
5 channel adapters implementing core.Channel.
Telegram, Discord, WhatsApp, Email, Web SSE. Each owns a JID prefix. Forward/reply metadata. Media placeholders. Trigger translation.
mime shipped
Magic-bytes MIME detection. Attachment pipeline.
Built-in signatures for 12 formats (JPEG, PNG, GIF, WEBP, AVI, WebM, MP4, OGG, FLAC, MP3, PDF, ZIP). RIFF subtype detection. Extension fallback. Handler interface for pluggable processing.
ipc shipped
File-based IPC watcher. Request/reply + legacy fire-and-forget.
Agent writes requests, gateway dispatches by type, writes replies. Atomic via tmp+rename. Non-root groups restricted to registered JIDs.
scheduler shipped
SQLite-backed task runner. Cron, interval, or once.
Polls due tasks every 60s. Cron via robfig/cron. Two context modes: group (injected as user message, shared session) or isolated (fresh session, results reported on next message).
diary shipped
YAML frontmatter diary annotations for agent context.
Reads recent .md files from group diary/. Extracts summary from YAML frontmatter. Returns XML annotations with age labels.
mountsec shipped
Mount allowlist validation.
Paths must be absolute, resolve symlinks, exist on host, be under an AllowedRoot. Blocks .ssh, .gnupg, .env, credentials, private keys.

Also: core/ (types, Channel interface), runtime/ (docker abstraction, orphan cleanup), groupfolder/ (path resolution), logger/ (slog JSON init), cmd/arizuko/ (CLI entrypoint).

Shipped Features

Container Isolation

Every agent runs in its own docker container. No shared process space. 8 volume mount types with security validation. Skills seeded on first spawn.

5 Channel Adapters

Telegram (long poll), Discord (WebSocket), WhatsApp (Signal protocol), Email (IMAP IDLE + SMTP), Web (SSE). Each implements the same Channel interface. Enabled by config presence.

Action Registry

14 actions with JSON schemas and authorization. Messaging, task scheduling, group management, session control, delegation. Root vs non-root access control.

MCP Sidecars

Per-group MCP servers as sidecar containers. Unix socket transport. Resource limits (memory, CPUs). Gateway manages full lifecycle.

Hierarchical Routing

5-tier routing rules: command, pattern, keyword, sender, default. Parent groups delegate to children. Same-world authorization. Max depth 3.

Signal-Driven IPC

File-based request/reply. Atomic writes via tmp+rename. SIGUSR1 wakes agent immediately. 500ms fallback poll.

MIME Pipeline

Magic-bytes detection for 12 formats. No external dependencies. Pluggable handler interface for attachment processing (transcription, OCR, etc).

48 Tests

Comprehensive test coverage across 8 packages: actions, channels, mime, router, store. Table-driven tests. Concurrent SSE tests. Magic-bytes detection tests with real headers.

Mythology

Arizuko is the goddess of the undead ants — the colony that cannot be killed. Each ant is an agent: isolated, purposeful, replaceable. When one falls, the goddess resurrects it. She routes work through the gateway and speaks through channels. The colony scales because the goddess composes, not controls.

Build & Run

$ make build && make image && make agent 
# create an instance
arizuko create foo

# edit config
vim /srv/data/arizuko_foo/.env

# register main group
arizuko group foo add tg:-123456789

# start
arizuko run

One binary. SQLite for state. Docker for agent isolation. No external dependencies beyond docker.

Agent context

ARIZUKO is a multitenant Claude agent gateway written in Go. It polls messaging channels (Telegram, Discord, WhatsApp, Email, Web SSE) for new messages, routes them to containerized Claude agents via docker, and streams responses back to users. 17 packages, ~9,800 LOC, 122 tests. Named after the goddess of the undead ants. Key components: gateway (message loop), container (docker spawn, sidecars), queue (per-group concurrency, circuit breaker), router (5-tier routing rules), store (SQLite WAL, 12 tables), channels (5 adapters), actions (14 actions, JSON schemas), ipc (file-based SIGUSR1), scheduler (cron/interval/once), mime (magic-bytes detection), diary (YAML annotations), mountsec (allowlist validation). Evolved from Kanipi (TypeScript). v1.0.0 adds permission tiers (4 levels), scheduled task context modes, IPC actions with authorization. v0.0.2 added channels, actions, MIME pipeline, and comprehensive test suite.
